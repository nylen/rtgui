The autoescape tag accepts two controls:

true or false
(default is true if not provided) These either turn variable auto-escaping on or off for the contents of the tag, regardless of the global setting.
escape-type
optionally include "js" to escape variables safe for JavaScript

For the following contexts, assume:

some_html_output = '<p>Hello "you" & \'them\'</p>';

So the following:

{% autoescape false %}
  {{ some_html_output }}
{% endautoescape %}

{% autoescape true %}
  {{ some_html_output }}
{% endautoescape %}

{% autoescape true "js" %}
  {{ some_html_output }}
{% endautoescape %}

Will output:

<p>Hello "you" & 'them'</p>

&lt;p&gt;Hello &quot;you&quot; &amp; &#39;them&#39; &lt;/p&gt;

\u003Cp\u003EHello \u0022you\u0022 & \u0027them\u0027\u003C\u005Cp\u003E

For more information on how the autoescape tag filters variable output, see the escape filter documentation.
